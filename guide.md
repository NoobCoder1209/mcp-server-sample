you'll have the demo running in about 90 seconds.

## What this repo is

A minimal **Model Context Protocol** server in TypeScript that wraps the Hacker News
public API. Three tools (`hn_top_stories`, `hn_get_story`, `hn_search`) over a
JSON-RPC stdio transport. No auth, no env vars, no persistent state. Designed to drop
into Claude Desktop or be exercised by the included probe script.

The README at the repo root is the buyer-facing pitch. **This document** is the
contributor / runner-of-the-demo guide.

## Prerequisites

You need three things on your machine. None of them are exotic.

| Tool       | Minimum version | Why                                              | How to install                                                                                  |
| ---------- | --------------- | ------------------------------------------------ | ----------------------------------------------------------------------------------------------- |
| **Node.js** | 20 LTS          | Runtime. The MCP SDK targets Node 20+.           | `brew install node@20` (macOS) · `nvm install 20` (any) · or download from nodejs.org           |
| **pnpm**    | 9.x             | Package manager pinned via `packageManager`.     | `corepack enable && corepack prepare pnpm@9.15.4 --activate`, or `npm install -g pnpm@9.15.4`   |
| **git**     | any             | Cloning the repo.                                | Almost always already installed; otherwise `brew install git` or your distro's package manager. |

You do **not** need Claude Desktop to verify the demo. The probe script is a hermetic
substitute. You **do** need Claude Desktop if you want to actually wire the server into
a model conversation; that's covered in [Wiring into Claude Desktop](#wiring-into-claude-desktop).

The CI runs Node 20 + pnpm 9.15.4. Anything that works in CI will work on your machine
with the same versions.

## Run the demo end-to-end

Five commands.

```bash
# 1. Clone the repo
git clone https://github.com/NoobCoder1209/mcp-server-sample.git
cd mcp-server-sample

# 2. Install dependencies (uses the committed lockfile — deterministic)
pnpm install --frozen-lockfile

# 3. Compile TypeScript to dist/ (also makes dist/index.js executable)
pnpm build

# 4. Run the test suite (11 unit tests + 3 integration tests against the built server)
pnpm test

# 5. Run the live demo: spawns dist/index.js, performs the JSON-RPC handshake,
#    lists the three tools, and calls hn_search for "Anthropic" against real HN.
pnpm probe
```

That's the demo. Steps 4 and 5 both prove the server works; step 5 also confirms
the live HN API is reachable from your machine.

## Verifying the demo actually worked

### After `pnpm test`

You should see:

```text
 ✓ test/tools.smoke.test.ts (11 tests)
 ✓ test/probe.integration.test.ts (3 tests)

 Test Files  2 passed (2)
      Tests  14 passed (14)
```

If `test/probe.integration.test.ts` reports `3 skipped` instead of `3 passed`, you
forgot to run `pnpm build` first — `dist/index.js` doesn't exist, so the integration
tests skip themselves. That's intentional; just run `pnpm build` and re-run.

### After `pnpm probe`

You should see, in order:

1. The ready banner on stderr: `mcp-server-sample 0.1.0 ready on stdio`
2. An `--- initialize ---` block showing `serverInfo.name: "mcp-server-sample"` and
   version `0.1.0`.
3. A `--- tools/list ---` line that prints exactly: `hn_top_stories, hn_get_story, hn_search`.
4. A `--- tools/call hn_search { query: "Anthropic", limit: 5 } ---` block followed by
   `HN search results for "Anthropic" (5):` and five numbered story entries with author,
   points, comments, date, and URL.

If you see all four blocks and the result count is 5 (it can occasionally be fewer if
HN drops some hits, but it should always be > 0), the demo worked.

## Environment variables and secrets

**There are none.** The Hacker News public APIs (Firebase + Algolia) are unauthenticated
and free. There are no API keys to manage, no `.env` to populate, and `.env.example` is
intentionally empty (kept for consistency with the rest of the portfolio).

If you fork this and add an authenticated upstream, the conventional place to add
secrets would be a `.env` file in the repo root (already gitignored via `.env.*`) and
an `import "dotenv/config"` near the top of `src/index.ts`. None of that is wired up
here because nothing needs it.

## Wiring into Claude Desktop

Claude Desktop's config file lives at:

| OS      | Path                                                                |
| ------- | ------------------------------------------------------------------- |
| macOS   | `~/Library/Application Support/Claude/claude_desktop_config.json`   |
| Windows | `%APPDATA%\Claude\claude_desktop_config.json`                       |
| Linux   | Not supported by Claude Desktop                                     |

Add an `mcpServers` entry pointing at your built `dist/index.js`. Replace the
`<absolute-path-to-mcp-server-sample>` placeholder with the actual absolute path to
your clone:

```json
{
  "mcpServers": {
    "hn": {
      "command": "node",
      "args": ["<absolute-path-to-mcp-server-sample>/dist/index.js"]
    }
  }
}
```

Then **fully quit and relaunch** Claude Desktop (Cmd+Q on macOS, right-click tray →
Quit on Windows). The MCP slider in the conversation input confirms the server loaded.
Ask Claude something like *"search HN for Anthropic"* — it will pick `hn_search` and
return live results.

If the slider doesn't appear, see [Common failure modes](#common-failure-modes-and-fixes)
below.

## Common failure modes and fixes

### `pnpm: command not found`

You don't have pnpm installed. The package manager is pinned to pnpm 9.15.4 via the
`packageManager` field in `package.json`. Two ways to install:

```bash
# Easiest (Node 20+ ships corepack)
corepack enable
corepack prepare pnpm@9.15.4 --activate

# Or via npm
npm install -g pnpm@9.15.4
```

### `pnpm install` fails with `ERR_PNPM_FROZEN_LOCKFILE_NEEDS_UPDATE` or similar

You're running a different pnpm major than the one that produced the lockfile. The repo
expects pnpm 9.x. Run the corepack commands above to pin to 9.15.4 and try again.

### `Cannot find module '/absolute/path/dist/index.js'` (in Claude Desktop logs)

You forgot to run `pnpm build`, or you typed the wrong absolute path in
`claude_desktop_config.json`. Fix:

```bash
cd /absolute/path/to/mcp-server-sample
pnpm build
ls dist/index.js     # confirm it exists
```

Then reopen Claude Desktop's config, double-check the path, save, and **fully quit and
relaunch** Claude Desktop (a window-level reload is not enough).

### Probe / integration test hangs or times out against HN

The Hacker News APIs are reliable but sometimes slow. The client has an 8-second
per-request timeout that translates into a friendly `isError: true` response. If you
see `HN API request timed out`, your network is the suspect — try `curl https://hacker-news.firebaseio.com/v0/topstories.json | head -c 100` to confirm
connectivity, then re-run.

### `pnpm test` reports `3 skipped` for `probe.integration.test.ts`

You ran `pnpm test` without `pnpm build`. Run `pnpm build && pnpm test` and the
integration tests will execute.

### "no tools" or empty `tools/list` response

The server registers tools at startup. If `tools/list` returns `{ tools: [] }`, you're
likely running a stale `dist/`. Rebuild:

```bash
pnpm build
```

### `console.log` corrupted the JSON-RPC stream

You added a `console.log` somewhere in `src/`. ESLint blocks this with the `no-console`
rule (only `console.error` is allowed in `src/`); the build will fail before this can
ship. If you're inside a long debug session, use `console.error` and read it on stderr.

### CI is red on a PR but `pnpm test` passes locally

Likely cause: you committed before running `pnpm build`. CI runs `install → lint →
typecheck → build → test`, so a build break (e.g. a TypeScript error) shows up there
even if your IDE cached an older type set. Run `pnpm build` locally to confirm.

## Where to go next

- Read `docs/architecture.md` for the design rationale and layer-by-layer breakdown.
- Read `PLAN.md` for the full execution plan that drove the build (research → design →
  build → tests → polish, with a peer-review gate on every PR).
- Edit `src/tools.ts` to add a fourth tool. The pattern is identical: add a zod
  field-record to `src/schemas.ts`, an HN client function if needed in `src/hn-client.ts`,
  and a `server.registerTool(name, def, safe(handler))` call in `src/tools.ts`.
